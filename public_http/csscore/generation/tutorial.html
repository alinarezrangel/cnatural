Tutorial

Getting started

Generation is a CSS3 based framework for HTML5, with a minimal ECMAScript 5/6 API for general purposes. The system requisites for using Generation are:

  • CSS3 support. All major browsers can view a Generation page without any trouble.
  • HTML5 support. All major browsers support HTML5.

The syntax of Generation is easy to learn, but first, you need download Generation 1.

Get Generation 1 (CSS3 Core)

Unzip the file in your page root's directory, after this, you can use Generation with this code (insert it in your head tag):

<link href="generation/css/generation.css" rel="stylesheet"/>
<script src="generation/js/generation.js"></script>

And you are ready for make a hello world page.

The "Hello World"

Create a directory named "helloworld" and unzip the Generation 1 framework in it.

After this, create a file named "index.html" in it and copy the next code in the file:

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8/>
    <title>Hello World</title>
    <link href="generation/css/generation.css" rel="stylesheet"/>
    <script src="generation/js/generation.js"></script>
  </head>
  <body>
    <main class="flexible center color-white">
      <h1>Hello World</h1>
    </main>
  </body>
</html>

In Generation 1, the colors have the syntax:

  • yellow
  • grey

The colors are: (1) the color, (2) the color with more intensity and (3) the color with more darkness.

Blocks and containers

In Generation 1, are 3 types of blocks:

  • boxes: A simple block.
  • containers: A block with padding and margin.
  • flexibles: A flexible box (see CSS3 flex-boxes).

The classes for they are:

  • boxes: box, card, grid.
  • containers: container, group, section.
  • flexibles: row, flexible, fcontainer, button-group.

Generation 1 have sub-types of blocks:

  • box: A simple box.
  • card: A box with shadow.
  • grid: A multiple collumn grid.
  • container: A simple container.
  • group: A collection of boxes, containers or flexibles.
  • section: A container with margin.
  • row: A simple flexible.
  • flexible: A simple flexible as a row.
  • fcontainer: As a container, but with flexible properties.
  • button-group: A sub-class of group, for buttons.

You can see the aspect of they in the demonstration file.

The usage is:

<element class="block-type classes">...

Where: element is any HTML element to style, block-type is any of the sub-types of blocks and classes are any other Generation 1 class.

12 columns fluid grid

Generation 1 have a multiple columns grid based on CSS3 flexboxes, but for general purposes, is safe use 12 of it.

By default, all items in a row have the same width, but you can change it with the classes fn where n is a number between 1 and 12 (inclusive).

You can control the width of an item with these classes, a f1 item have a default width, a f2 item have the double of default width, but if all items in the row have a f2 class, their really have a default width, not the double!.

For a very good explanation of this unexpected result, you can see the official CSS3 FlexBox reference.

In standards terms, the "f" class sets the CSS flex property, and the "o" class sets the order property (o1, o2, o3, o4 ...).

You can set a lot of properties with classes, for example: "wrap" and "no-wrap", "align-stretch" or "self-start", etc. The basic flexible classes are:

  • fn: Sets the flex width
  • on: Sets the order
  • self-start: (on columns) align this item to the top of the flex
  • self-end: (on columns) align this item to the bottom of the flex
  • self-stretch: (on columns) align this item to the height of the flex
  • self-center: (on columns) align this item to the center of the flex
  • align-class: aligns all items. class is start, end, stretch of center
  • direction-row: set the flex direction to row (default in rows)
  • direction-column: set the flex direction to column (default with center)
  • wrap: wraps the items
  • no-wrap: never wrap items

Borders

For set the border of an element, you can use the border class, but if you not need all borders use it's sub-classes:

  • border: sets all borders
  • border-vertical: sets only the top and bottom borders
  • border-horizontal: sets only the left and right borders
  • no-border: removes all borders
  • no-border-side: removes the side border, the sides are top, bottom, left or right

For sets the border color, use any of the border-color-name classes, the names are normal color names, and support light and dark colors:

class="card border border-color-aqua"
class="card border-horizontal border-color-dark-green"
class="card border-vertical border-color-light-red"

For sets the border width, use the "bs-n" classes, where n is: 1, 2, 4, 6, 8, 16, 32 or 64 (measures in px).

class="card bs-4 border border-color-aqua"
class="card bs-2 border-horizontal border-color-dark-green"
class="card bs-32 border-vertical border-color-light-red"

Others and utilities

Padding

"padding-n" where n is 1, 2, 4, 8, 16, 32 or 64.

Remove the padding with "no-padding".

Margin

"margin-n" where n is 1, 2, 4, 8, 16, 32 or 64.

Remove the margin with "no-margin".

Width and height

"total-width" set the width to 100vh (screen width). "width-block" set the width to 100% (all containers width).

"total-height" set the height to 100vh (screen height). "width-block" set the height to 100% (all containers height).
